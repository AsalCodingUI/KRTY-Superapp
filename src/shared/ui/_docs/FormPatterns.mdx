# Form Patterns

Guidelines for building accessible and consistent forms.

## Form Layout

Use a vertical stack for form fields on mobile, and grid layouts on desktop where appropriate. Labels should always be visible.

```tsx
<form className="space-y-4">
    <div className="space-y-2">
        <Label htmlFor="email">Email</Label>
        <Input id="email" type="email" placeholder="name@example.com" />
    </div>
    
    <div className="space-y-2">
        <Label htmlFor="role">Role</Label>
        <Select id="role">
            <SelectItem value="admin">Admin</SelectItem>
            <SelectItem value="user">User</SelectItem>
        </Select>
    </div>

    <div className="pt-4">
        <Button type="submit">Save Changes</Button>
    </div>
</form>
```

## Action Buttons

Place primary actions (Save, Submit) on the right of the form footer, or full width on mobile. Secondary actions (Cancel) should be to the left of the primary action.
